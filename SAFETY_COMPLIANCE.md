## 2. Secrets & Credentials Protection

### Zero Secret Leak Policy
- **No Hardcoded Keys**: The repository contains **zero** hardcoded API keys, authorization tokens, cloud passwords, or private SSH keys.
- **Placeholder Standards**: Any references to HuggingFace tokens (`hf_...`), OpenAI keys (`sk-...`), or database credentials utilize clear placeholders (e.g. `YOUR_HF_TOKEN`) inside instructions or documentation.
- **Git Guardian Audits**: The [.gitignore](file:///d:/Domain/.gitignore) is pre-configured to automatically exclude any environment variables files (`.env`, `.env.local`, `.env.*.local`) from version control.

### SRE Security Design
All generated YAML manifests and scripts generated by the studios enforce best-practice SRE security models:
- **Kubernetes Secret Bindings**: Generated configurations load sensitive environment variables (such as HuggingFace tokens) using Kubernetes `Secret` namespaces and `valueFrom.secretKeyRef` bindings, preventing plain-text tokens inside workloads.
- **Docker Compose Environment variables**: Environment variables are mapped using compose config variables or secure local paths rather than plain-text configuration strings.

## 3. Machine Learning & Model Licensing

- **No Weight Redistribution**: The portfolio **does not download, package, or distribute** large/small language model weights (such as Meta Llama, Google Gemma, or Microsoft Phi).
- **Client-Side Model Fetching**: The generated bootstrapping scripts instruct the deployment environment to download base model weights directly from official sources (such as `ollama.com` or official Docker registries). 
- **Model Terms of Use**: End users accept the respective model's Terms of Use (e.g., Llama 3 Community License or Gemma Terms of Use) when executing the deployment scripts in their own private cluster environments.

---

## 4. Code Protection & Deployed Assets

- **Vite Compilation & Minification**: During build pipeline operations (`npm run build`), Vite compiles all custom files. It runs a minification pass (via Esbuild) that:
  - Compresses source code size.
  - Strips comments, annotations, and formatting.
  - Obfuscates variables and structures.
- **Static Site Security**: Because the site is fully static (HTML/JS), it exposes **zero server-side database endpoints**, eliminating typical SQL injection, remote code execution (RCE), or backend credential breach risks.
